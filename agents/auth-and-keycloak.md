# Auth and Keycloak

## Scope

- Covers: OAuth2, Keycloak realm generation, token flow, JWKS URLs, client IDs, bootstrap admin, and
  runtime Keycloak user provisioning.
- Read when: changing auth config, Keycloak generation, auth clients, UI login, or wallet user
  setup.
- Excludes: Canton rights semantics except where they interact with JWT users.
- Supporting docs: `test/integration/auth_confidence_test.ts`.

## What this subsystem is

OAuth2 with Keycloak is the only supported auth mode. Keycloak serves realm imports generated by the
project, web UIs authenticate through public clients, service clients use client credentials, and
runtime YAML-defined users are also created in the relevant validator realm.

## Main modules

- `src/generator/keycloak.ts`: realm JSON generation and bootstrap admin constants.
- `src/generator/splice.ts`: JWKS URL and auth config consumed by Splice apps.
- `src/api/auth.ts`: `TokenManager` and `createAuthHeader()`.
- `src/api/keycloak-admin.ts`: runtime Keycloak Admin client.
- `src/types/config.ts`: realm and client ID helpers.
- `src/docker/containers.ts`: Keycloak container command, env, health check, web UI auth env.

## Working rules

- Realm names are title-cased validator names; `sv` resolves to `SV`.
- Use `resolveRealmName()` when handling arbitrary validator names that may include `sv`.
- Keycloak and Splice realm naming must agree or wallet APIs return 401.
- Default username equals password for built-in validator users and YAML-defined users.
- The persistent master realm admin comes from `auth.keycloak.admin/password`.
- A temporary bootstrap admin is created for startup and deleted after Keycloak is ready.

## Critical gotchas

- `BOOTSTRAP_ADMIN_USERNAME` is `localnet-internal-bootstrap-do-not-use`.
- `KC_BOOTSTRAP_ADMIN_PASSWORD` is a fixed sentinel; it is not a user secret.
- Keycloak runs `start-dev --import-realm --proxy-headers=forwarded`.
- `KC_HTTP_ENABLED=true` and `KC_HOSTNAME_STRICT=false` are intentional for local HTTP and proxy
  use.
- Master realm `sslRequired: 'none'` is intentional for Docker network admin calls.
- A username-to-`sub` protocol mapper is required because Splice derives ledger users from JWT
  claims.
- Access tokens are short-lived (`300` seconds); long integration flows must refresh or request new
  tokens.

## Editing guidance

- When changing clients, update web UI env generation, token manager tests, auth confidence tests,
  and docs together.
- When changing default users or realms, verify both API token flows and browser login flows.
- When changing bootstrap admin behavior, verify master realm admin access after startup and
  deletion of the bootstrap user.
